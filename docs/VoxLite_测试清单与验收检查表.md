# 和乐轻音（Holoo VoxLite）测试清单与验收检查表

## 1. 使用说明

- 本文档用于研发自测、测试回归、里程碑验收三类场景。
- 范围覆盖：功能正确性、体验指标、稳定性、兼容性、发布门禁。
- 执行原则：先主链路、再异常链路、最后性能与兼容。

## 2. 测试分层与责任

| 层级 | 目标 | 主要责任 |
| --- | --- | --- |
| 单元测试 | 验证状态机、路由规则、错误码映射正确 | 开发 |
| 集成测试 | 验证跨模块协作与降级回退链路正确 | 开发 + 测试 |
| 回归测试 | 验证三类场景体验与文案一致性 | 测试 + 产品 |
| 发布验收 | 验证双架构、性能、包体、门禁项 | 测试 + 研发负责人 |

## 7. 里程碑验收检查表（Checklist）

## M1 可跑通验收

- [ ] 可完成“按住 Fn 说 -> 松开写入”主链路。
- [ ] 三类场景至少各完成 1 次成功转写与注入。
- [ ] 状态机完整迁移并可追踪日志。
- [ ] 错误码可覆盖权限、识别、注入核心错误。

## M2 可稳定验收

- [ ] 首次引导流程与“稍后配置”分支完整可用。
- [ ] 权限缺失、模型不可用、注入失败三类回退可用。
- [ ] 超时策略与重试机制可用。
- [ ] 性能指标达到 P50/P95 目标阈值。
- [ ] 连续 30 分钟使用无焦点异常抢占。

## M3 可发布验收

- [ ] Intel 与 Apple Silicon 全量通过回归。
- [ ] 测试报告中 P0 缺陷全部关闭。
- [ ] 安装包体积落入 5MB-10MB 区间。
- [ ] 发布说明、风险清单、回滚预案齐备。

## 8. 缺陷处理与准出规则

- 缺陷分级：
  - P0：主链路不可用、文本丢失、崩溃、无法恢复；
  - P1：核心体验明显受损但可绕过；
  - P2：一般问题，不阻断发布。
- 准出标准：
  - M1 阶段允许 P2 未关闭，P0/P1 必须关闭；
  - M2/M3 阶段 P0 必须为 0，P1 需有明确豁免并经评审；
  - 涉及“文本丢失”的问题一票否决，不得发布。
