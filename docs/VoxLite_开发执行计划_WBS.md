# 和乐轻音（Holoo VoxLite）开发执行计划（WBS）

## 1. 文档说明

- 目标：将已确认的 M1/M2/M3 路线拆解为可执行任务、输入输出、完成定义。
- 适用角色：产品、设计、研发、测试。
- 约束前提：
  - macOS Menu Bar App；
  - Swift 6 + SwiftUI + AppKit；
  - 0 第三方依赖优先；
  - 核心链路“按住 Fn 说 -> 松开写入”必须始终可用。

## 2. WBS 总览

| 编号 | 工作包 | 目标 | 对应里程碑 |
| --- | --- | --- | --- |
| WP-01 | 工程基线与架构 | 建立分层、协议、状态机、错误码与埋点骨架 | M1 |
| WP-02 | 输入链路实现 | 完成 Fn 监听与录音会话管理 | M1 |
| WP-03 | 识别与清洗链路 | 完成转写、上下文识别、三场景清洗路由 | M1 |
| WP-04 | 输出注入与回退 | 完成注入、失败回退、文本不丢失保障 | M1/M2 |
| WP-05 | 引导与运行态体验 | 完成首次引导、状态反馈、场景切换 | M2 |
| WP-06 | 稳定性与性能优化 | 达成时延、CPU、内存、超时恢复目标 | M2 |
| WP-07 | 质量体系与发布准备 | 完成测试矩阵、双架构验证、发布收敛 | M3 |

## 3. 工作包明细

## WP-01 工程基线与架构

- 目标：
  - 建立 Input/Core/Output/System 四层边界；
  - 建立统一结果对象 `success/errorCode/latencyMs`；
  - 建立统一状态机 `Idle/Recording/Processing/Injecting/Done/Failed`。
- 主要任务：
  - 定义核心协议与数据结构；
  - 定义错误码分层（权限、识别、清洗、注入、超时）；
  - 定义日志等级与埋点事件名称规范。
- 输出物：
  - 架构说明与模块边界文档；
  - 协议与模型定义文件；
  - 状态迁移图与埋点字典。
- 完成定义（DoD）：
  - 任意主流程请求可关联 `sessionId`；
  - 关键状态迁移具备可追踪日志与打点；
  - 模块依赖方向符合分层约束。

## WP-02 输入链路实现

- 目标：
  - 长按右 Fn 开始录音，抬起结束录音并进入处理态；
  - 录音状态反馈可感知且低延迟。
- 主要任务：
  - 基于 `CGEventTap + flagsChanged` 构建 Fn 状态机；
  - 增加防抖与误触判定（短于 500ms 可标记误触）；
  - 建立 `AVAudioEngine` 采集会话及中断恢复。
- 输出物：
  - 输入监听模块；
  - 录音会话管理模块；
  - 输入侧监控指标（触发成功率、误触发率）。
- 完成定义（DoD）：
  - 触发状态反馈 <100ms；
  - 不破坏系统快捷键行为；
  - 录音会话可正确开启、结束、取消与恢复。

## WP-03 识别与清洗链路

- 目标：
  - 默认端侧识别可用；
  - 场景化清洗符合沟通/开发/写作三类输出风格。
- 主要任务：
  - 实现 `SFSpeechRecognizer` 转写流程（优先 `requiresOnDeviceRecognition=true`）；
  - 实现上下文识别：前台 App + 焦点输入角色；
  - 实现三路清洗模板与路由决策逻辑；
  - 模型不可用时降级“仅转录不清洗”。
- 输出物：
  - 转写服务；
  - Context Engine；
  - 清洗路由器与风格标签输出。
- 完成定义（DoD）：
  - 三场景输出文本风格可区分且可预测；
  - 降级路径可自动执行；
  - 关键处理时延可测量。

## WP-04 输出注入与回退

- 目标：
  - 将结果可靠写入当前焦点输入区域；
  - 失败时保证文本不丢失。
- 主要任务：
  - 实现剪贴板快照替身流程（备份/写入/粘贴/恢复）；
  - 实现注入失败回退（复制剪贴板 + 提示手动粘贴）；
  - 实现注入结果分类与错误码映射。
- 输出物：
  - 注入模块；
  - 回退模块；
  - 注入成功率与回退比例指标。
- 完成定义（DoD）：
  - 正常路径可稳定写入；
  - 失败路径文本可获取且不丢失；
  - 50ms 级剪贴板恢复稳定执行。

## WP-05 引导与运行态体验

- 目标：
  - 对齐原型中的首次引导与运行态核心交互；
  - 保持低打扰、可预期的状态反馈。
- 主要任务：
  - 实现首次引导步骤：欢迎、麦克风、辅助功能、语音识别、演练；
  - 实现“继续下一步/稍后配置”两种分支；
  - 实现场景切换与对应风格展示；
  - 实现异常提示卡片（权限缺失、模型不可用、注入失败）。
- 输出物：
  - 引导界面；
  - 状态栏菜单与运行态反馈；
  - 文案与提示规范。
- 完成定义（DoD）：
  - 首次引导可在 30 秒内完成首轮录入；
  - 运行态不出现多余阻断弹窗；
  - 关键提示信息可执行、可理解。

## WP-06 稳定性与性能优化

- 目标：
  - 达成 PRD/TSD 指标；
  - 提升长期使用稳定性。
- 主要任务：
  - 构建端到端时延采集（P50/P95）；
  - 性能优化：空闲 CPU、录音峰值 CPU、空闲内存；
  - 增加超时与重试机制（单次 >3s 提示重试）；
  - 完成焦点异常与权限变化场景稳态处理。
- 输出物：
  - 性能测试报告；
  - 超时策略与重试策略实现；
  - 稳定性问题闭环列表。
- 完成定义（DoD）：
  - P50 <1.0s，P95 <1.8s；
  - 空闲 CPU <3%，空闲内存 <150MB；
  - 连续 30 分钟使用无焦点抢占异常。

## WP-07 质量体系与发布准备

- 目标：
  - 构建可发布质量门禁；
  - 完成双架构发布前验证。
- 主要任务：
  - 单元测试：状态机、路由规则、错误码映射；
  - 集成测试：权限变化、切 App、注入失败、降级链路；
  - 回归测试：沟通/开发/写作三场景；
  - 构建 Universal Binary 与安装包体积控制。
- 输出物：
  - 测试报告与缺陷清单；
  - 发布检查报告；
  - 发布候选包与回滚预案。
- 完成定义（DoD）：
  - Intel/Apple Silicon 核心链路均通过；
  - 发布门禁项全部通过；
  - 包体落在 5MB-10MB 目标区间。

## 4. 里程碑门禁（Gates）

| Gate | 对应里程碑 | 通过条件 |
| --- | --- | --- |
| G1 | M1 可跑通 | 主链路跑通；三场景至少一轮成功写入；错误码可追踪 |
| G2 | M2 可稳定 | 权限引导、失败回退、超时重试齐备；体验与性能达标 |
| G3 | M3 可发布 | 双架构验证通过；测试报告通过；发布包与回滚预案齐备 |

## 5. 建议协作节奏

- 每个工作包结束必须做一次评审：产品验收 + 技术验收 + 测试验收。
- 每次评审产出三项结论：是否过 Gate、阻塞问题、下一轮优先级调整。
- 所有线上可观测指标在 M2 前必须打通，M3 前完成可视化看板。
