# 和乐轻音 (Holoo VoxLite) - 产品需求及技术方案规格说明

## 一、文档目标与适用范围

- **文档目标：** 同时服务产品、设计、研发、测试四类角色，确保“体验目标”与“工程实现”一一对应。
- **MVP 形态：** macOS 状态栏常驻应用（Menu Bar App），暂不包含 iOS 端。
- **目标用户：** 高频文字输入者（沟通、开发、写作三类场景）。
- **核心承诺：** 轻量、低打扰、可预期、端侧优先、隐私可解释。

---

## 二、产品需求文档 (PRD)

### 1. 愿景与定位

- **所属组织：** 和乐 (Holoo)，坚持“让工作生活更和乐美满”。
- **产品名称：** 和乐轻音 / Holoo VoxLite。
- **产品定位：** 一款极轻量（目标安装包 <10MB）、按住即说即放即写的系统级语音录入助手。
- **设计原则：**
  - **无侵入：** 不改变用户既有输入习惯。
  - **低认知：** 全流程依赖短时状态反馈，不引入复杂操作面板。
  - **高确定性：** 用户始终知道“是否正在录音”“是否已写入”。

### 2. UX 目标与可用性指标

- **启动可理解性：** 首次引导后，用户可在 30 秒内完成首次录入。
- **交互可感知性：** 录音开始/结束状态反馈延迟 <100ms。
- **端到端时延：** 松开触发键到文本完成写入 P50 <1.0s，P95 <1.8s。
- **文本可用率：** 无需手工修改即可直接发送/粘贴的比例 >85%（按周统计）。
- **误触容忍：** 误触发率 <2%（以录音时长 <500ms 且用户撤销记为误触）。

### 3. 用户场景与关键流程

#### 3.1 核心用户故事

1. **无感触发：** 用户在任意输入场景长按右侧 `Fn` 键，状态栏图标进入“录音中”态。
2. **自然表达：** 用户按口语表达，不要求刻意断句。
3. **智能清洗：** 松开 `Fn` 键后，系统完成转录、场景识别、文本清洗。
4. **无感上屏：** 清洗后的文本写入当前焦点输入区域，用户继续编辑或直接发送。

#### 3.2 交互状态机

- **Idle：** 待机，不占焦点。
- **Recording：** 采集音频，展示微弱动效。
- **Processing：** 录音结束，执行识别与清洗。
- **Injecting：** 执行文本写入。
- **Done / Failed：** 成功提示轻反馈；失败给出可恢复提示（重试、复制到剪贴板）。

状态迁移必须可追踪并可打点，便于后续体验优化。

### 4. 功能范围与非目标

#### 4.1 MVP 功能范围

- **全局触发：** 监听右侧 `Fn` 键按下/抬起。
- **语音转文本：** 优先端侧识别，网络不可用时仍可用。
- **场景化清洗：**
  - **沟通场景（微信/Slack/飞书）：** 精简、口语友好、单段输出。
  - **开发场景（Xcode/Terminal/VSCode）：** 支持命名风格转换与简短提交语气。
  - **写作场景（Notion/Pages）：** 书面化、段落化输出。
- **文本注入：** 将结果可靠写入当前焦点输入区域。

#### 4.2 非目标（MVP 不做）

- 自定义热键录制器与复杂快捷键映射。
- 多语言混说自动翻译。
- 云端账号体系与历史记录同步。
- 长音频文件离线批量转写。

### 5. 权限、隐私与合规

- **系统权限：** 麦克风、辅助功能（Accessibility）、语音识别。
- **隐私策略：** 默认端侧处理；不持久化原始音频；不上传可识别个人信息。
- **可解释性：** 在设置页明确告知“采集内容、处理位置、保留时长（如无则标明不保留）”。

### 6. 失败处理与降级策略

- **权限缺失：** 提供一步直达系统设置与可视化修复指引。
- **模型不可用：** 降级为“仅转录不清洗”。
- **注入失败：** 自动回退到“复制到剪贴板 + 提示手动粘贴”。
- **超时策略：** 单次处理超过 3s 触发超时提示并允许重试。

### 7. 验收标准（PRD 级）

- 连续使用 30 分钟不出现输入焦点异常抢占。
- 常见三类场景（沟通/开发/写作）均可稳定完成“按住说 -> 松开写”。
- 在 Apple Silicon 与 Intel 机型上均可完成核心流程，且无阻塞性卡顿。

## 三、技术方案规格说明 (TSD)

### 1. 技术选型与边界

- **Bundle ID：** `ai.holoo.voxlite`。
- **开发语言：** Swift 6 + SwiftUI + AppKit（菜单栏与系统事件场景）。
- **依赖策略：** 0 第三方依赖，优先系统原生框架。
- **平台范围：** 首发 macOS（建议最低版本 macOS 15+，按实际 API 可用性校准）。
- **包体目标：** 安装包 5MB-10MB。

### 2. 分层架构

- **Input Layer：** 全局事件监听、音频采集。
- **Core Layer：** 语音识别、上下文探测、文本清洗路由。
- **Output Layer：** 文本注入、回退写入、状态反馈。
- **System Layer：** 权限管理、日志与指标采集、配置中心。

各层仅通过明确协议通信，避免跨层直接调用。

### 3. 核心模块实现

#### 模块 A：全局事件拦截（Global Event Tap）

- 使用 `CGEventTap` 监听 `flagsChanged`，构建 `Fn` 键状态机。
- `keyDown(Fn)` 触发 `startRecording()`；`keyUp(Fn)` 触发 `stopRecordingAndProcess()`。
- 需规避与系统组合键冲突，保证原生快捷键行为不被破坏。

#### 模块 B：语音识别（Speech to Text）

- 使用 `SFSpeechRecognizer` 与 `AVAudioEngine`。
- 设定 `requiresOnDeviceRecognition = true` 作为默认策略。
- 对识别会话设置超时与中断恢复机制（来电、设备切换、权限变化）。

#### 模块 C：场景感知（Context Engine）

- 使用 `NSWorkspace.shared.frontmostApplication` 获取前台 App。
- 使用 `AXUIElement` 获取焦点输入元素类型。
- 输出结构化上下文：`{bundleId, appCategory, inputRole, locale}`。

#### 模块 D：文本清洗（Foundation Models Route）

- 输入：`rawTranscript + context`。
- 输出：`cleanText + confidence + styleTag`。
- 采用路由模板：沟通/开发/写作三类 Prompt，确保行为可预测。

#### 模块 E：文本注入（Clipboard Injection）

- 采用“剪贴板快照替身”：
  1. 备份当前剪贴板。
  2. 写入清洗结果。
  3. 模拟 `Cmd + V`。
  4. 约 50ms 后恢复原剪贴板。
- 若粘贴失败，执行回退路径并提示用户。

### 4. 关键接口契约（示意）

- `startRecording() -> SessionID`
- `stopRecording(sessionId) -> AudioBuffer`
- `transcribe(audio) -> TranscriptResult`
- `resolveContext() -> ContextInfo`
- `cleanText(transcript, context) -> CleanResult`
- `injectText(cleanText) -> InjectResult`

所有结果对象必须包含 `success`、`errorCode`、`latencyMs` 字段，便于监控与排障。

### 5. 性能与资源预算

- **常驻内存：** 目标 <150MB（空闲态）。
- **空闲 CPU：** 平均 <3%。
- **录音态 CPU 峰值：** 目标 <25%（短时峰值可放宽）。
- **端到端时延：** 与 PRD 指标保持一致并按版本追踪。
- **兼容性：** 提供 Universal Binary，覆盖 Intel 与 Apple Silicon。

### 6. 可观测性与质量保障

- **核心埋点：** 触发成功率、识别成功率、注入成功率、端到端时延分位数、回退比例。
- **日志分级：** `debug/info/warn/error`，默认不记录原始语音与全文文本。
- **测试策略：**
  - 单元测试：状态机、路由规则、错误码映射。
  - 集成测试：权限变化、切 App、粘贴失败回退链路。
  - 手工回归：三类场景 + 双架构机型（Intel/Apple Silicon）。

### 7. 里程碑建议

- **M1（可跑通）：** 完成按住说-松开写主链路。
- **M2（可稳定）：** 完成权限引导、失败回退、性能优化。
- **M3（可发布）：** 完成兼容性验证、指标看板、发布包体收敛。
