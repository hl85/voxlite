# VoxLite 高优体验升级开发计划

> 基于 `handless_vs_voxlite_源码对比报告.md`，以**体验改进优先**为原则制定。

## 设计原则
- **体验改进优先**：聚焦用户最直接感知的交互、反馈、兼容性和性能。
- **保持轻量原生**：不盲目引入重型框架，坚持 Swift 6 + SwiftUI + AppKit，保持 VoxLite 的低系统表面积优势。
- **架构平滑演进**：在现有分层架构（Input/Core/Output/System）基础上，引入 Coordinator 和 Manager 模式，避免 `AppViewModel` 进一步膨胀。

---

## 阶段一：核心交互与状态反馈升级 (交互体验)
**目标**：解决当前单一“按住-松开”模式的局限，提供更灵活的触发方式和更清晰的视觉反馈。

1. **引入 `TranscriptionCoordinator` (架构与交互升级)**
   - 从 `AppViewModel` 中剥离录音生命周期控制，专门处理输入事件与业务阶段的竞态。
   - **新增触发模式**：支持 `Hold`（按住录音，松开结束）和 `Toggle`（按一下开始，再按一下结束）。
   - **新增取消机制**：支持在录音或处理过程中主动取消（如按 `Esc` 键放弃本次输入）。
2. **新增悬浮状态反馈 `RecordingOverlay` (视觉体验升级)**
   - 开发一个轻量级的独立悬浮窗（Overlay），在录音和处理阶段提供明显的视觉反馈（如录音波形、处理进度、状态提示）。
   - 解决纯菜单栏 UI 在全屏应用或多显示器场景下“不知道是否在录音”的视觉盲区痛点。

---

## 阶段二：文本注入兼容性增强 (输出体验)
**目标**：解决单一剪贴板注入在部分特殊应用（如终端、远程桌面、特定编辑器）中可能失效或表现不佳的问题。

1. **抽象注入策略层 `TextInjectionStrategy`**
   - 将现有的 `ClipboardTextInjector` 改造为默认策略实现。
2. **新增备选注入实现**
   - 实现 `DirectTypingInjector`：基于 `CGEvent` 模拟键盘逐字敲击，解决剪贴板被拦截或不支持粘贴的场景。
3. **注入策略配置 UI**
   - 在设置面板中暴露“注入方式”选择，允许用户根据自身常用软件环境全局切换注入策略。

---

## 阶段三：模型生命周期与性能优化 (性能体验)
**目标**：提升首次语音识别的响应速度，同时优化长期后台运行的内存占用。

1. **引入 `ModelManager`**
   - 负责本地模型和远端 Provider 的统一生命周期管理，将模型作为长期资产管理。
2. **实现预加载与空闲卸载机制**
   - **Preload (预加载)**：应用启动或唤醒时，提前加载端侧模型进内存，消除首次唤醒的冷启动延迟（降低 P50 延迟）。
   - **Idle Unload (空闲卸载)**：长时间无语音输入时，自动释放模型内存，保持 VoxLite 的轻量级后台占用。

---

## 阶段四：后处理与技能定制产品化 (定制体验)
**目标**：让用户能够根据自己的工作流（如写代码、写邮件、日常聊天）自定义清洗规则和 Prompt。

1. **Prompt 与技能管理层**
   - 扩展 `TextCleaner` 和 `ContextResolver`，支持读取用户自定义的 Prompt 模板，而不仅限于硬编码的规则。
2. **自定义技能 UI**
   - 在设置面板新增“技能/后处理”管理界面，允许用户增删改查自定义 Prompt。
   - 支持将特定 Prompt 绑定到特定的应用上下文（通过 BundleID 映射）。

---

## 执行规范 (遵循 Workspace Rules)
1. **测试驱动**：每个阶段必须先写失败测试（状态机/路由/回退），再写最小实现，最后重构并保持测试全绿。
2. **性能门槛**：任何改造不得拖慢主链路，提交前必须通过性能门槛断言（P50 < 1.0s、P95 < 1.8s）。
3. **隐私约束**：新增的 Overlay 和历史功能不得持久化原始音频，日志不得记录全文内容。
